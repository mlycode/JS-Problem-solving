Doubly Linked Lists - Data Stuctures

Overview

  • Like a singly linked list, but every node points to both the next and previous node
  • More flexibility but also uses more memory (two points instead of one)

Code

Creating the Node and Doubly linked list classes

  • Almost identical to singly linked lists but the Node class has an extra property of this.prev

Creating the .push(value) method - add a node to the end of the list

  • first conditional defined for when the list is empty
  • else, the old tail has its .next assigned as the new node, and the new node has its .prev set to the old tail, finally the tail is reassigned as the new node
  • List length is incremented adn the list is returned

Creating the .pop() method - removing a node form the end of the list

  • Removes the old tail's .prev connection and the previous node from the tail (new tail) .next connection also removed

Creating the .shift() method - remove a node from the start of the list

  • For empty list, return undifined
  • For 1 node list, sets the head and tail to null
  • else the connections are set to null to remove the head form the list, reassigning the .next of the old head as the new head

Creating the .unshift(value) method - insert a new node at the start of the list

  • creates newNode variable
  • sets the current head's .prev property to point to the newNode
  • newNode's .next points to this.head, then the head is reassigned to the newNode

Creating the .get(index) method - get the value from a specified index

  • Conditional that checks if the index of the value to be found is greater than or less than the midpoint of this.length
  • Conditional used to decide if the pass through the array should start from the head or the tail (whichever is shortest) making the get method more efficient compared to a singly linked list

Creating the .set(index, value) method - replace the value of a specified node

  • uses the get() method to find the node to change

Creating the .insert(index, value) method - add a node at a specified index

  • Uses the get method to find the node before the insert index and then connects the .next and .prev properties for the nodes before and after
  • conditionals use push and unshift methods to insert at the start or end of the list

Creating the remove(index) method - remove a node from a specified index

  • uses the get method to find the node to be removed
  • the nodes previous and next have their .next and .prev properties changed and the removed node's properties of next and prev are set to null

Big O time complexity

  • Insertion is O(1) - same as singly linked lists
  • Removal is O(1) - is always constant unlike singly linked lists (worst case when removing from the end of the list which is O(n))
  • Searching is O(n) - same as singly linked lists but technically half of singly linked lists (just simplifies to O(n))
  • Access - O(n)
