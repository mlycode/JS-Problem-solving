Binary Search Trees

Overview

  • A Tree is a data structure that consists of nodes in a parent/child relationship
  • Trees are non-linear, there are multiple paths
  • A node can only point to a child element (move down from a single root node, not across)

Terminology

  • Root - the top node in a tree
  • child - a node directly connected to another node when moving away from the root
  • Parent - converse notion of a child
  • Siblings - a group of nodes with the same parent
  • Leaf - node with no children
  • Edge - the connection between one node to another

Uses for Trees

  • HTML DOM; elements and nested elements (children)
  • Network routing - finding shortest routes
  • AI
  • Folder structure
  • JSON

Types of Trees (Many)

  • Trees
  • Binary Trees
  • Binary Search Trees - excel at searching; each node can only have two children and are sorted (store comparable data)
    • child nodes to the left of a parent are less than the parent
    • Makes inserting and searching quicker compared to unsorted trees

Code

Creating the classes

Creating the insert() method - insert a node in the correct position (depending on value)

  • Creating a new node, newNode variable. Checking first if a root exists
  • if it does exist, while true (breaking only for a return statement) if greater than the current parent node, move to the right and check again
  • only return when a left or right child node does not exist, then set it to newNode

Creating the find(value) method - search if a value exists within a tree

  • While there is a current value (not equal to null) and the found variable is false, conditionals run
  • if nothing is found (current is false) then false is returned, else something was found and true is returned

Big O of BST

  • Insertion, O(logn) - Doubling the number of nodes only increases the number of node levels by 1 (increases the number of steps by one)
  • Searching, O(logn) - However, every child could be on one side leading to O(n)
