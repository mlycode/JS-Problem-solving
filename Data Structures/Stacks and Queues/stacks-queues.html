Stacks and Queues

Stacks

Overview

  • It is a LIFO data structure (last in first out)
  • In a Stack, the last element added will be the first element removed from the stack
  • Used for managing function invocations (call stack)
  • Used for undo / redo and routing (history)

Code

There is more than one way to implement a stack

Using an Array to create a Stack

  • An array using .push() to add onto the end of the array and .pop() to remove the end of the array satisfies LIFO
  • .shift() and .unshift() can also do this, but remember adding and removing from the start of an array has O(n) time complexity due to requirement of re-indexing the array elements
  • An array has many methods and indices which aren't needed for a stack

Creating a Stack using a Singly Linked list

  • Stack class only needs two methods to add and remove the last item in the list
  • push and pop remove and add from the start of the list/stack (because actual push and pop are not constant time for a singly linked list)

Big O of Stacks

  • Insertion, O(1) - ensured that the push method was O(1)
  • Removal, O(1)
  • Searching and Access, O(n) - nature of linked lists, could be slightly improved with doubly linked lists but this isn't important for stacks

Queues

Overview

  • It has FIFO data structure (first in first out)
  • Used for background tasks, uploading resources, printing queues

Code

Creating a Queue using an Array

  • use .shift() to remove from the start of the queue (of course requires re-indexing O(n)) and .push() to add to the end of the queue
  • Instead could use .unshift() to add to the 'end' of the queue and .pop() to remove from the start of the queue but just switches the case where re-indexing is required

Creating a Queue using a List

  • Adding to the end of the list(enqueue) and removing(dequeue) from the start
  • Do this to ensure constant time complexity O(1)

Big O of Queues

  • Insertion and Removal, O(1)
  • Searching and Access, O(n)
