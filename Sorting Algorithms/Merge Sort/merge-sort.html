Merge Sort Algorithm

Merge, Quick and Radix Sort?

  • Bubble, Selection and Insertion Sort don't scale well (take longer for larger arrays) - These have quadratic time complexity
  • Merge, Quick and Radix Sort improve the time complexity to O(nlogn), which is better than quadratic

Merge Sort

Overview:

  • A combination of merging and sorting
  • Exploits the fact that arrays of 0 or 1 element are always sorted
  • Works by decomposing an array into smaller arrays of 0 or 1 elements, then building up a sorted array by merging the smaller arrays in sorted order
    • The single element arrays are sorted, making merging the sorted arrays easier than an unsorted array

Code

Run mergeSort([array of unsorted values]) in the console

  • Uses recursive code that splits the array into single element arrays before merging the arrays in sorted order
  • Has O(n) Space Complexity
  • Has O(nlogn) Time complexity
    • O(logn) from splitting the array (number of decompositions), O(n) from comparing with the merge function resulting in O(nlogn)
